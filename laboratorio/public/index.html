Formularios


  • Type Text: Este tipo de input toma un valor de "texto", por lo que crea una sola línea de entrada de texto.

  • Type Password: Como su nombre lo indica, una entrada con un tipo de contraseña crea una contraseña. Es automáticamente invisible para el usuario, a menos que sea manipulado por JavaScript.

  • Type Email: Este tipo de input toma un valor de "correo electrónico", por lo que crea una sola línea de entrada de texto.

  • Type Submit: Este tipo de input toma un valor de "enviar", por lo que crea un botón de envío.

  • Type Button: Este tipo de input toma un valor de "botón", por lo que crea un botón.

  • Type Fecha: Es posible que se haya registrado en un sitio web donde solicitó la fecha de un evento determinado. La web probablemente usó una entrada con el valor de tipo establecido "date" para lograr esto.

  • Type Color: Este tipo de input toma un valor de "color", por lo que crea una sola línea de entrada de texto.

  • Type Radio: Este tipo de input toma un valor de "radio", por lo que crea una sola línea de entrada de texto.

Ocultar Elementos


  • Visibility: controla la visibilidad del elemento, visible o no, pero no afecta a como el navegador representa la caja. Por tanto, visibility: hidden esconde el elemento pero el espacio que ocupa sigue representándose.

  • Display: cada elemento en HTML se puede circunscribir a un rectángulo, referido generalmente como box o caja. El comportamiento de esta caja se puede controlar a través de la propiedad display que le dice al navegador como debe representarla: en línea (valor inline), como un bloque (valor block), etc. El valor display: none le dice al navegador que no lo represente; el resultado es que el elemento queda escondido y el espacio que debería ocupar colapsado.
